Introduction to Grid

CSS Grid Basic Concepts

CSS Grid is a 2 dimensional system which uses an intersecting set of horizonal and diagonal lines to place and align its child elements. With Grid you have the ability to layer elements, create nested grids, adjust the positioning of entire grouped areas as the viewport resizes and add additional rows or columns when needed.

Elements can be positioned:

  • Explictly, using px values, fixed track sizes and precise locations.
  • Flexibly, utilizing the grid algorithm, spercentages and fractions to control the document flow.

To get started with Grid, first create a wrapper and change the display to grid, each direct descendant of the wrapper will then become a grid child.


     .wrapper
     display: grid;
      or
     display: inline-grid;

Creating Grid Tracks

Explict Grid Tracks

Grid Tracks can define rows and columns within the grid using: 


      grid-template-columns:
      grid-template-rows:

In the first example below, tracks are created by applying an individual value to each required column such as px/fr. The second example shows the use of repeat() notation to generate a specific number of equal sized rows. Both methods can be used on either rows or columns to create an explict grid


      grid-template-columns: 200px 1fr 2fr;
      grid-template-rows: repeat(3, 1fr);

Automatic Rows

Controlling Automatic Row Spacing

When items extend the specified grid or content needs to flow into new rows, grid will create new rows which are auto-sized to fit their content.
The following examples show how to control this default behaviour: 


      grid-auto-rows: 300px
      grid-auto-rows: minmax(200px, auto)

In the first example above, all rows will conform to the specified height of 300px as they are created. However, by using the minmax() function you can create a more flexible row that will stretch to accomodate the content (auto) whilst not allowing the row to shrink below a specific minimum height, which in this case is 200px.

Positioning Grid Items

Targeting Grid Lines

A simple way to position grid items is line-based placement, the below example shows a grid item using this method to define its position within the grid container


      grid-column-start: 1
      grid-column-end: 3
      grid-row-start: 2
      grid-row-end: 3
  • Declaration 1. Will place the item at the start of the first vertical grid line
  • Declaration 2. States the item will end at the third vertical grid line, so it will span two columns in total from the column start position
  • Declaration 3. Will place the item at the start of the second horizonal grid line
  • Declaration 4. States the item will end at the third horizonal grid line, spanning one row in total from the row start position

Defining Gutters

Using Grid-Gap Properties

Gutters are easily created within the Grid Container, however browser support does vary and often will need to be vendor prefixed.
When using css grid it is always advisable to check support by visiting caniuse.com. The below shows four different examples of usage, however in addition to px, percentage or em units you are also able to use both rem and calc() functions to assign values. 


      grid-column-gap: 10px;
      grid-row-gap: 1em;
      grid-gap: 10%;
      grid-gap: 1em 2em;
  • Declaration 1. Uses a fixed px value to create 10px wide column gutters
  • Declaration 2. Assigns an em value to create 1em row gutters
  • Declaration 3. grid-gap or gap, will assign the specified value to both the row and column gutters
  • Declaration 4. Declaring two values on the grid-gap property will assign the first to the row gutter and the second to the column gutter
